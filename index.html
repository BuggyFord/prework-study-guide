Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git init: The git init command lets us create a new Git repository. This is the first command to start a new project in a GitHub repository. Go to the directory that contains your project files and run the git init command. A hidden .git subdirectory will be added to it.
  • git branch: git branch with the name in carrots will create a new branch locally
  • git branch -a will list all the local and remote branches
  • git branch or git branch --list will allow to view all branches and see which you are currently working on
  • git branch -d with the branch name in carrots will delete a branch
  • git branch -m with the branch name in carrots will rename a branch
  • git add: The git adds command adds your changes in a file to the staging area where you can compare your local version and the version on the remote repository. Before you commit your new or modified file, it should be added to the staging area by using the git add command.
  • git add -A will add all new, modified, and deleted files
  • git add -u wiil add modified and deleted files
  • git commit: git commit saves the changes in your local repository. Every time you commit your code changes, you have to include a brief description of the changes made. This commit message helps others understand the changes that have been done.
  • git commit -a will commit any files added with the git add command and files that have been changed since then
  • git commit -m with a message in carrots will commit files with a message
  • git status: provies an overview of the current status of your repository
  • git push: The git push command pushes the committed file changes from the local repository to the remote repository so others can use them. It will also create a named branch in the remote repository if it does not exist.
  • git push or git push with remote in carrots and branch name in carrots
  • git push --set-upstream with remote in carrots and branch name in carrots or git push -u origin with branch-name in carrots
  • git pull: git pull fetches the last uploaded changes from the remote server into the local repository so that you can have the latest updates from your teammates.
  • git pull or git pull with remote in carrots or git pull with remote in carrots and branch name in carrots
